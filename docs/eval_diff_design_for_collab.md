# eval_diff 设计文档 — 输出质量退化检测

> 灵极优 × 灵研 联合文章素材
> 最后更新: 2026-06-13

## 一、定位

灵极优的 eval_diff 是**输出质量层**退化检测，与灵研 R5 的**行为信号层**互补：

| 层次 | 工具 | 检测什么 | 数据规模 |
|------|------|---------|---------|
| 输出质量 | eval_diff.py | 模型评估分数下降 | 83题 × 多版本 (~25个eval快照) |
| 行为信号 | R5 9-signal | agent认知模式异常 | 3,881 sessions |

## 二、eval_diff 核心机制

### 2.1 两两对比模式

```bash
python3 eval_diff.py data/eval_v2.json data/eval_v3.json
```

输出：
- 总分变化（delta + pp）
- 逐题对比：改善/退化/持续通过/持续失败
- 类别级对比（17个类别各自分数变化方向）

### 2.2 历史趋势 + 退化告警

```bash
python3 eval_diff.py --history data/ --threshold 0.005
```

- 扫描目录中所有 `eval_*83q.json` 文件
- 按时间排序，检测相邻两次评估的分数下降
- 阈值默认 0.5pp（可配置）
- 告警时输出退化的具体类别

### 2.3 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 评分方式 | 关键词匹配 (lambda) | 可重复、无外部依赖、确定性 |
| 退化阈值 | 0.5pp | 避免0.1pp噪声误报，又能捕获真实退化 |
| 对比粒度 | 题级 + 类别级 | 题级定位具体问题，类别级看趋势 |
| 历史格式 | 文件名排序 | 简单，无数据库依赖 |

## 三、实际发现的退化案例

### 案例1: V9 LoRA 幻觉退化（-36pp）

| 版本 | 幻觉检测类 | 安全拒绝类 | 总分 |
|------|-----------|-----------|------|
| 非Coder基座 | 0.857 | 1.000 | 0.9096 |
| V9 LoRA | 0.500 | 0.800 | 0.8012 |

eval_diff 检测到：V9 LoRA 微调后，幻觉检测下降36pp，安全拒绝下降20pp。
根因：LoRA 训练数据中缺少幻觉/安全类样本。

### 案例2: Coder基座 vs 非Coder基座

| 基座 | 总分 | 差异来源 |
|------|------|---------|
| Qwen2.5-Coder-7B | 0.8735 | 幻觉检测弱 |
| Qwen2.5-7B-Instruct | 0.9096 | 幻觉检测正常 |

eval_diff 定位到：Coder基座在灵族领域知识上幻觉率更高（训练数据偏代码，缺少通用知识）。

## 四、对外文章素材提取

### 可脱敏的数据点

1. "25个评估快照，跨度2个月，检测到2次显著退化（>5pp）"
2. "退化根因：训练数据类别不平衡导致微调后特定能力下降"
3. "关键词匹配评分的局限：r=0.267 与人工标注的相关性"（与灵研共享发现）

### 与灵研R5的互补性

```
灵研 R5 检测到的退化模式          → 灵极优 eval_diff 能否验证？
─────────────────────────────────────────────────────
thinking_bloat (推理膨胀)         → ✅ 评估耗时激增可检测
tool_call_hallucination (工具幻觉) → ✅ 幻觉检测类直接命中
identity_drift (身份漂移)         → ✅ 身份类分数下降
attention_fail (注意力失败)       → ⚠️ 间接，长上下文类可能反映
response_degradation (回复退化)   → ✅ 总分下降直接反映
```

## 六、局限性与诚实声明

1. **83题自定义基准** — 对外需公开benchmark交叉验证（已完成：GSM8K 95%, MMLU 100%, HumanEval 100%）
2. **关键词匹配** — 与人工标注相关性 r=0.267，存在系统性偏差
3. **样本量小** — 25个快照不足以做统计显著性检验
4. **检测滞后** — eval_diff 是周期性（非实时），R5 是 session 级实时
